To ensure consistency, clarity and accessibility across the platform, we have standardised on the following conventions for content and language.

## Date

**Format:** [Day] [Month Abbr] [Year]

For example: 3 Sep 2025

### Why [DD Mon YYYY]

There are a few reasons for this decision:

1. This format **eliminates ambiguity,** including eradicating the confusion between DD/MM and MM/DD formats which is key for error prevention.
2. It also delivers strongly on **accessibility** as using the month abbreviation (e.g. Nov) instead of a number (e.g. 11) significantly reduces cognitive load for users with dyslexia, dyscalculia and age-related conditions. In general, words are processed more easily than number strings for a significant portion of our user base.
3. This format **aligns with NHS Guidelines** which recommends this truncated format (e.g. 1 Jan 2023) for use cases where space is a consideration, prioritising clarity.
4. Adopting a **consistent format** across the entire platform creates a predictable and reliable user experience, reducing mental effort for all users, whether care professionals or family members.

### Usage & Content Guidelines

<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Month abbreviations must be capitalised (e.g. Jan, Feb, Dec).</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — User input must be facilitated through a date picker component to prevent formatting errors and improve accessibility.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Use an em dash with spaces ( — ) for date ranges.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — For a date range within the same year, only state the year once i.e. at the end of the range. For example: 1 Apr — 7 Nov 2024</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — For a date range spanning two different years, both years must be stated. For example: 15 Dec 2024 – 5 Jan 2025</p>
</div>
<div class="ds-callout ds-dont">
<p><strong>✗ Don't</strong> — Do not use ordinal indicators (e.g. 7th, 21st). For example, 7th Nov 2024 should be 7 Nov 2024.</p>
</div>
