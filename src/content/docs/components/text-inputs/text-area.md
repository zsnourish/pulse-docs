A multi-line field for long-form, free-text input e.g. comments, descriptions. These typically include handles for vertical resizing. Can have a character limit, and may include helper text to guide the user.

## Usage & Content Guidelines

<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Use descriptive helper text below the field (e.g. "Passwords must be 8+ characters").</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Mark required fields with an asterisk (*) and add "Required" in error state.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Ensure use of focus states.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Placeholder text should hint at format (e.g. "Describe your issue in detail...").</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Auto-expand as the user types (avoid scrollbars within the field).</p>
</div>
<div class="ds-callout ds-dont">
<p><strong>✗ Don't</strong> — Avoid Optional labels, use asterisk (*) for required fields instead.</p>
</div>
<div class="ds-callout ds-dont">
<p><strong>✗ Don't</strong> — Avoid pre-filled fields unless data is highly predictable (e.g. country based on IP).</p>
</div>
<div class="ds-callout ds-dont">
<p><strong>✗ Don't</strong> — Avoid validating unless necessary (e.g., maximum length) with error message below the field.</p>
</div>
