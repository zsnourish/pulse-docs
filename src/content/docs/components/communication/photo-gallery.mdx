**Photo gallery component** is a grid display for a collection of images. Thumbnails are uniform in size, and clicking an image opens it in a lightbox or overlay view.

## Status

**Status tags** are used to quickly convey the state of an item (shared taxonomy with [Tags → Status](/components/tags/)): Attention, Error, Urgency, Success, General, Greyed.

### Usage & Content Guidelines

<div class="ds-callout-columns">
<div class="ds-callout-col">
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Maintain consistency: Always use the same colour for the same status across the entire application.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Pair with simple icons only when they meaningfully enhance comprehension.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Keep text short (1-3 words max).</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Apply consistently within a section (all tags either icon+text or text-only).</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Use for broader metadata (use dedicated tag sets where relevant).</p>
</div>
</div>
<div class="ds-callout-col">
<div class="ds-callout ds-dont">
<p><strong>✗ Don't</strong> — Do not overload with icons.</p>
</div>
<div class="ds-callout ds-dont">
<p><strong>✗ Don't</strong> — Do not use for actionable elements (tags are static, non-interactive).</p>
</div>
<div class="ds-callout ds-dont">
<p><strong>✗ Don't</strong> — Do not exceed one line (truncate or simplify labels as needed).</p>
</div>
</div>
</div>
