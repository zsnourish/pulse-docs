**Tags** (also referred to as badges) are a single-line static tag input for displaying short labeled items, supporting text and numbers.

## Basic

Available in the Accent colour set (Pink Clay, Lapis, Lavender, Topaz, Olive, Mulberry, Ice, Jade), each with an icon+text variant and a text-only variant.

## Status

**Status tags** are used to quickly convey the state of an item.

### Attention (Yellow)

- **Purpose:** Indicates a warning or pending state that requires future action but does not currently block progress.
- **When to Use:** Tagging items that are draft, expiring soon, awaiting action, or in a temporary state that needs monitoring.
- **Example Labels:** Draft, Unverified, Expiring Soon, Pending Review, Action Needed, Incomplete

### Error (Orange)

- **Purpose:** Indicates an active, blocking state that requires immediate resolution. The system or process is currently halted because of this.
- **When to Use:** Tagging items that are stuck, failed, or in an error state that is blocking workflow.
- **Example Labels:** Upload Failed, Sync Error, Verification Failed, Action Required, Conflict Detected, Processing Halted

### Urgency (Red)

- **Purpose:** Indicates a critical, active state with serious, often permanent, consequences. The action has already happened or the state is currently true.
- **When to Use:** Tagging items that are in a severe, high-stakes, or terminated state.
- **Example Labels:** Archived, Terminated, Access Revoked, Suspended, Critical Alert, Data Expunged

### Success (Green)

- **Purpose:** Indicates a positive, completed, or optimal state.
- **When to Use:** Tagging items that are active, verified, completed successfully, or in good standing.
- **Example Labels:** Active, Verified, Completed, Approved, Healthy, Online

### General (Blue)

- **Purpose:** Indicates a neutral, informational, or in-progress state. This is the default for passive statuses.
- **When to Use:** Tagging items that are processing, in a queue, under review, or simply for providing neutral metadata.
- **Example Labels:** Pending, In Review, Processing..., Scheduled, Informational, Default

### Greyed (Grey)

- **Purpose:** To indicate an inactive, unavailable, or not-yet-initiated state.
- **When to Use:** For items that are dormant, scheduled, canceled, or not applicable.
- **Example Labels:** Not Started, Scheduled, Canceled, Voided, N/A, Disabled
