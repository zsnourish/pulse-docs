## Usage & Content Guidelines

<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Always include a clear, concise label e.g. First Name.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Show real-time validation (e.g. Invalid email format). Use our semantic error colour and error message below the field.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — On mobile, ensure the correct keyboard type appears (e.g. numeric for phone numbers).</p>
</div>
<div class="ds-callout ds-dont">
<p><strong>✗ Don't</strong> — Do not use placeholders in place of labels. Only use placeholder text for hints if needed, and sparingly.</p>
</div>
