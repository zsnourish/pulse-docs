Buttons for triggering actions. Pulse defines three variants: Primary, Secondary, and Destructive.

## Primary

**Primary buttons** are the most important action on a screen (e.g. submitting a form, confirming an action). These should be used sparingly, usually only one per screen.

### Usage & Content Guidelines

<div class="ds-callout-columns">
<div class="ds-callout-col">
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Use action-oriented labels starting with strong verbs e.g. Save, Confirm, Submit.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Be concise, use 1-2 words and avoid long phrases.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Always prioritise the primary action, with Primary CTA placement on the right.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — For consistency, use the same label wording for similar actions across Nourish.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Use icons where beneficial (not always needed).</p>
</div>
</div>
<div class="ds-callout-col">
<div class="ds-callout ds-dont">
<p><strong>✗ Don't</strong> — Avoid vague terms e.g. use Create Account instead of Continue. Avoid negativity e.g. Cancel, not Ignore.</p>
</div>
</div>
</div>

### Usage & Content Guidelines

<div class="ds-callout-columns">
<div class="ds-callout-col">
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Use only for actions which significantly alter a system in a negative way, usually permanently e.g. something exists now but will be deleted. Example labels include: delete, revoke, end, discharge.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Use action-oriented clear labels like Delete, Revoke.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Be concise, use 1-2 words and avoid long phrases.</p>
</div>
<div class="ds-callout ds-do">
<p><strong>✓ Do</strong> — Use icons where beneficial (not always needed).</p>
</div>
</div>
<div class="ds-callout-col">
<div class="ds-callout ds-dont">
<p><strong>✗ Don't</strong> — Do not use for 'Cancel', 'Back' or 'Close'. 'Cancel' is typically a neutral action i.e. does not change any data or commit to anything. Typically, it simply closes the current view.</p>
</div>
<div class="ds-callout ds-dont">
<p><strong>✗ Don't</strong> — Do not use for 'Archive', which is a reversible action and simply a change of status.</p>
</div>
</div>
</div>
